# WMS — Система управления складом

Учебный проект по предмету «Разработка клиент-серверных приложений».

## Стек
- Python + Flask
- SQLite
- HTML + CSS + JavaScript (fetch)

## Структура проекта

```
wms/
  app.py                  # Flask-приложение (сервер)
  warehouse.db            # SQLite база (создается скриптом)
  requirements.txt        # зависимости Python
  database/
    schema.sql            # создание таблиц
    seed.sql              # тестовые данные
    init_db.py            # скрипт инициализации БД
  templates/
    base.html             # общий шаблон с навигацией
    index.html            # главная страница
    products.html         # товары
    warehouses.html       # склады
    stock.html            # остатки
    shipments.html        # отгрузки
    shipment_detail.html  # детали отгрузки
  static/
    css/
      style.css
    js/
      main.js
```

## Запуск

### Обычный локальный запуск

1. Установить зависимости:
   ```
   pip install -r requirements.txt
   ```

2. Инициализировать базу данных:
   ```
   python database/init_db.py
   ```

3. Запустить сервер:
   ```
   python app.py
   ```

4. Открыть в браузере: http://localhost:5000

## Запуск через Docker

1. Собрать Docker-образ:
   ```
   docker build -t wms-app .
   ```

2. Запустить контейнер:
   ```
   docker run -p 5000:5000 wms-app
   ```

3. Открыть в браузере: http://localhost:5000

Примечание:
- при сборке Docker-образа тестовая база данных создается автоматически;
- приложение внутри контейнера запускается на порту `5000`.

## Переменные окружения

Проект поддерживает простые переменные окружения:

- `PORT` — порт запуска Flask-приложения;
- `SECRET_KEY` — секретный ключ Flask для flash-сообщений.

Пример локального запуска с указанием порта:

```
set PORT=5000
python app.py
```

## Как показать проект преподавателю

Самый простой вариант:

1. Показать локальный запуск через Docker.
2. Открыть страницы товаров, складов, остатков и отгрузок.
3. Продемонстрировать, что данные берутся из SQLite-базы.
4. Показать, что проект можно поднять одной инструкцией из README.

## Публичный запуск

Если нужна ссылка на проект, проще всего использовать платформу, которая умеет собирать приложение из `Dockerfile`.

Для учебного проекта достаточно такого сценария:
- загрузить проект в GitHub;
- подключить репозиторий к платформе деплоя;
- дождаться сборки контейнера;
- открыть выданную ссылку.

Важно: в текущем простом варианте SQLite-база создается внутри Docker-образа. Для демонстрации и сдачи этого достаточно.
